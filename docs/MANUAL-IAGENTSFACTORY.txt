===============================================================
1. O QUE ESTE SISTEMA E
===============================================================

O IAgentsFactory e uma fabrica de software com IA e memoria persistente.

Na pratica, ele junta 3 camadas:

1. ISGT
   - Templates, agents, skills, prompts e automacao de setup.

2. Knowledge Hub da Factory
   - Banco SQLite local em %USERPROFILE%\.iagents-factory\knowledge.db
   - Guarda solucoes aprendidas, projetos registrados, export/import,
     metricas de reuso e economia de tokens.

3. MCP Graph Workflow
   - Servidor HTTP + dashboard visual.
   - Focado em grafo de tarefas, PRD, backlog, insights e visualizacao.
   - Hoje ele NAO e o banco principal do Knowledge Hub da Factory.


===============================================================
2. MODELO MENTAL CORRETO
===============================================================

Use assim:

A. Factory = memoria operacional e reuso de conhecimento
   - search
   - search-cross
   - capture
   - import
   - export
   - stats

B. MCP Graph = servidor web + dashboard visual
   - serve
   - dashboard
   - visualizacao de grafo, backlog, insights

C. Dashboard nativo da Factory = visao direta do knowledge.db
  - KPIs
  - alertas
  - uso por projeto
  - top solucoes e capturas recentes

Resumo simples:

- Quer salvar, buscar, reutilizar e medir conhecimento?
  Use a Factory.

- Quer interface visual, grafo e dashboard web?
  Use o MCP Graph.

10. CICLO CORRETO DE USO DO KNOWLEDGE HUB
===============================================================

Fluxo recomendado no dia a dia:

PASSO 1 - Buscar antes de perguntar para IA externa

  .\iagents-factory.ps1 search "calculo de ROI"

Se encontrar algo util:
  - reutilize a solucao

Se nao encontrar:
  - pergunte ao agente externo
  - depois capture a resposta na Factory


PASSO 2 - Capturar conhecimento novo

Modo interativo:

  .\iagents-factory.ps1 capture

Modo por arquivo .solution.md:

  .\capture-pipeline.ps1 -FromFile .\config\_example.solution.md -Project IAgentsFactory


PASSO 3 - Buscar novamente depois

  .\iagents-factory.ps1 search "roi"


PASSO 4 - Ver metricas

  .\iagents-factory.ps1 stats


===============================================================
11. FORMAS DE CAPTURA SUPORTADAS
===============================================================

A. Captura interativa manual

  .\iagents-factory.ps1 capture

Quando usar:
  - voce acabou de receber uma solucao boa do agente externo
  - quer cadastrar com mais controle


B. Captura por arquivo .solution.md

  .\capture-pipeline.ps1 -FromFile arquivo.solution.md -Project MeuProjeto

Quando usar:
  - voce quer um fluxo reproduzivel
  - vai montar um acervo de solucoes versionadas


C. Captura em lote

  .\capture-pipeline.ps1 -Batch .\solutions\ -Project MeuProjeto

Quando usar:
  - migracao inicial de conhecimento
  - importacao de varias solucoes de uma vez


D. Captura por watch/clipboard

  .\capture-pipeline.ps1 -Watch -Project MeuProjeto

Quando usar:
  - voce quer monitorar clipboard e capturar rapidamente


===============================================================
12. FORMATO CORRETO DO ARQUIVO .solution.md
===============================================================

Exemplo:

  ---
  domain: financial
  pattern: calculation
  language: java
  framework: spring-boot
  agent: claude-sonnet
  quality: 0.9
  tags: roi, investment, calculation
  ---

  ## Prompt
  <pergunta original>

  ## Solution
  <codigo ou resposta>

  ## Summary
  <resumo curto para busca>

Campos importantes:

- domain
  dominio de negocio

- pattern
  tipo de solucao

- language
  linguagem principal

- framework
  stack ou framework

- quality
  score inicial da solucao

- tags
  palavras para busca e classificacao


===============================================================
13. COMO BUSCAR CONHECIMENTO
===============================================================

Busca local no projeto atual:

  .\iagents-factory.ps1 search "roi"

Busca em outros projetos registrados:

  .\iagents-factory.ps1 search-cross "roi"

Comportamento esperado:

- search
  - procura no Knowledge Hub inteiro
  - considera o contexto do projeto atual

- search-cross
  - tenta trazer conhecimento de outros projetos registrados
  - se o projeto atual estiver identificado, exclui os matches do proprio projeto

===============================================================
19. TROUBLESHOOTING
===============================================================

PROBLEMA: dashboard nao sobe

Verifique:
  - MCP Graph foi buildado?
  - existe pasta dist no repositorio mcp-graph-workflow?

Comando:
  Set-Location "C:\Users\AR CALHAU\source\repos\mcp-graph-workflow"
  npm run build


PROBLEMA: search nao encontra nada

Verifique:
  - a solucao foi realmente capturada?
  - stats mostra pelo menos 1 solucao?
  - o arquivo .solution.md esta com formato correto?


PROBLEMA: export gera 0 solucoes

Verifique:
  - stats mostra solucoes armazenadas?
  - quality_score das solucoes esta acima de 0.5?


PROBLEMA: GitNexus mostra warning

Hoje isso pode acontecer no Windows e nao bloqueia o dashboard.
Se o objetivo for usar o dashboard e o Knowledge Hub, voce pode seguir mesmo assim.
